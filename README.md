
# 🐬 Pool CRM - Система управления плавательным бассейном

Полноценная CRM-система для управления детскими спортивными группами, расписанием, посещаемостью и коммуникацией с родителями через VK бота.

## ✨ Возможности

### 👑 Администратор
- Управление группами, тренерами, детьми
- Обработка заявок на зачисление (одобрение/отклонение)
- Управление переводами между группами
- Ведение фотогалереи бассейна и тренеров
- Контроль посещаемости
- Генерация расписания тренировок

### 🏊 Тренер
- Просмотр своих групп
- Отметка посещаемости на тренировках
- Создание запросов на перевод детей (в т.ч. зачисление новых детей без группы)
- Просмотр расписания

### 👨‍👩‍👧 Родитель
- Добавление/редактирование данных детей
- Просмотр расписания и посещаемости
- Привязка VK аккаунта для уведомлений
- Подача заявки на зачисление (даже без регистрации)

### 🤖 VK Бот
Команды для привязанных родителей:
- `дети` - список детей с группами
- `сегодня` - сегодняшние тренировки
- `завтра` - завтрашние тренировки
- `неделя` - тренировки на неделю
- `уведомления вкл/выкл` - управление оповещениями
- `помощь` - справка

## 🚀 Запуск

### 1. Клонирование и установка
```
git clone https://github.com/your/pool-crm.git
cd pool-crm
pip install -r requirements.txt
```

### 2. Инициализация БД
```
python seed_database.py   # Заполнение тестовыми данными
```

### 3. Запуск
```
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

### 4. Доступ к системе
- **Веб-интерфейс:** http://127.0.0.1:8000

# 🗄️ МОДЕЛИ ДАННЫХ (database.py)

## 📌 Перечисления (Enums)

### `UserRole`
Роли пользователей в системе.

| Значение | Описание |
|----------|----------|
| `parent` | Родитель (доступ к детям, расписанию, посещаемости) |
| `coach` | Тренер (доступ к своим группам, отметкам) |
| `admin` | Администратор (полный доступ) |

### `EnrollmentStatus`
Статус зачисления ребёнка в группу.

| Значение | Описание |
|----------|----------|
| `active` | Активное зачисление (ребёнок посещает группу) |
| `frozen` | Заморожено (временный перерыв, оплата не идёт) |
| `completed` | Завершено (ребёнок отчислен или переведён) |
| `waiting_list` | Лист ожидания (нет мест в группе) |

### `TrainingStatus`
Статус тренировки.

| Значение | Описание |
|----------|----------|
| `scheduled` | Запланирована (предстоит) |
| `completed` | Проведена (прошла) |
| `cancelled` | Отменена (не состоялась) |

### `AttendanceStatus`
Статус посещаемости ребёнка на тренировке.

| Значение | Описание |
|----------|----------|
| `present` | Присутствовал |
| `absent_sick` | Отсутствовал по болезни |
| `absent_family` | Отсутствовал по семейным обстоятельствам |
| `absent_no_reason` | Отсутствовал без уважительной причины |
| `frozen_skip` | Пропуск из-за заморозки (автоматически) |

### `ApplicationStatus`
Статус заявки на зачисление.

| Значение | Описание |
|----------|----------|
| `pending_parent_verification` | Ожидает подтверждения родителя |
| `new` | Новая заявка (требует рассмотрения) |
| `on_review` | На рассмотрении (админ просматривает) |
| `approved` | Одобрена (созданы родитель, ребёнок, зачисление) |
| `rejected` | Отклонена |
| `waiting_list` | В листе ожидания (группа переполнена) |

## ⏰ Таблица временных слотов (`TimeSlotDB`)

Хранит регулярное расписание групп (повторяется каждую неделю).

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | Integer | Первичный ключ |
| `group_id` | Integer | Внешний ключ → `groups.id` |
| `day_of_week` | Integer | День недели (0=пн, 1=вт, ..., 6=вс) |
| `start_time` | Time | Время начала тренировки |
| `end_time` | Time | Время окончания тренировки |

**Уникальность:** `(group_id, day_of_week, start_time)` - у группы не может быть двух одинаковых слотов.

**Связи:**
- `group` → группа (`GroupDB`)
- `trainings` → тренировки (`TrainingDB`)

| `status` | TrainingStatus | Статус тренировки |

**Связи:**
- `group` → группа (`GroupDB`)
- `time_slot` → временной слот (`TimeSlotDB`)
- `attendances` → отметки посещаемости (`AttendanceDB`)
