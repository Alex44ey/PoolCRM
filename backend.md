## 🔧 ВСПОМОГАТЕЛЬНЫЕ ФУНКЦИИ

| Функция | Назначение | Возвращает |
|---------|------------|------------|
| `get_current_user(request, db)` | Извлекает пользователя из cookie сессии | Словарь с данными пользователя или `None` |
| `get_user_object(request, db)` | Возвращает объект ORM пользователя из БД | `ParentDB`, `CoachDB` или `None` |
| `calculate_age(birthdate)` | Вычисляет возраст в годах на основе даты рождения | `float` (возраст) |
| `get_group_enrollment_count(group_id, db)` | Считает количество активных детей в группе | `int` |
| `check_parent_access(child_id, parent_id, db)` | Проверяет, принадлежит ли ребёнок родителю | `bool` или `HTTPException` |
| `check_coach_access(group_id, coach_id, db)` | Проверяет, принадлежит ли группа тренеру | `bool` или `HTTPException` |
| `check_admin_access(user_role)` | Проверяет, является ли пользователь админом | `bool` или `HTTPException` |
| `get_child_or_404(child_id, db)` | Находит ребёнка по ID или возвращает 404 | `ChildDB` |
| `get_group_or_404(group_id, db)` | Находит группу по ID или возвращает 404 | `GroupDB` |
| `get_coach_or_404(coach_id, db)` | Находит тренера по ID или возвращает 404 | `CoachDB` |
| `get_training_or_404(training_id, db)` | Находит тренировку по ID или возвращает 404 | `TrainingDB` |

### ВВЕДЕНИЕ

> **Привет, я Алексей.**
>
> Я работал над **backend-частью** проекта — файлом `main.py`. Это веб-сервер на **FastAPI**, который обрабатывает все запросы, работает с базой данных и общается с фронтендом.
>
> Мой код отвечает за: аутентификацию, ролевую модель, управление детьми, группами, тренерами, тренировками, посещаемостью, заявками и переводами.

---

### АРХИТЕКТУРА И ТЕХНОЛОГИИ

> **Технологический стек:**
> - **FastAPI** — современный асинхронный фреймворк на Python
> - **SQLAlchemy** — ORM для работы с базой данных
> - **Jinja2** — шаблонизатор для HTML
> - **SQLite** — лёгкая база данных
>
> **Как это работает:**
> ```
> Браузер → FastAPI (main.py) → База данных → HTML ответ
>          ↓
>     VK Бот (vk_bot.py)
> ```
> 
> **Ролевая модель.**
> У нас три роли: `admin`, `coach`, `parent`. От роли зависит, какие данные видит пользователь.
>

### ДЕМО

> **Сейчас я покажу, как работает наше приложение:**
>
> 1. Вход как администратор — вижу дашборд со статистикой.
> 2. Просмотр заявок — одобряю новую заявку на зачисление.
> 3. Отметка посещаемости как тренер.
> 4. Генерация VK кода как родитель.
> 5. Фотогалерея — загрузка и просмотр фото.
> 6. Страница 404 с дельфином 🐬
